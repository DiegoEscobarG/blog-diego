Guía Completa: Subir un Proyecto a GitHub

Aprende a subir tu proyecto a un repositorio digital paso a paso — desde la instalación de Git y los comandos necesarios hasta llegar a tu repositorio en la nube.

Requisitos previos: Tener Git instalado y una cuenta en github.com

Git es un sistema de control de versiones distribuido, gratuito y de código abierto. Permite gestionar el historial de cambios en tu código, volver a versiones anteriores y trabajar en equipo sin sobrescribir el trabajo de otros.

Tu carpeta local → git init → git add → git commit → Crear repo en GitHub → git push → ✓ En la nube
Pasos
1

Instalar Git y verificar la instalación

Descarga Git desde git-scm.com/downloads según tu sistema operativo e instálalo con las opciones por defecto. Una vez instalado, abre tu terminal y verifica:

terminal
git --version
Resultado esperado
git version 2.53.0
Tip: El número de versión puede variar. Lo importante es que no arroje un error.
2

Configurar tus credenciales

Git necesita saber quién eres antes de registrar cualquier cambio. Esto se hace una sola vez en tu computadora.

terminal
git config --global user.name "Tu Nombre" git config --global user.email "tu@email.com" # Para confirmar que se guardaron correctamente: git config --list
Resultado esperado (al final)
user.name=Tu Nombre
user.email=tu@email.com
Importante: El email debe ser el mismo con el que creaste tu cuenta de GitHub. Si no coinciden, tus commits no aparecerán vinculados a tu perfil.
3

Abrir la terminal en tu carpeta de proyecto

Navega hasta la carpeta donde está tu proyecto. Tienes dos opciones:

Opción A — Terminal

Navega usando el comando cd hasta llegar a tu carpeta.

Opción B — Explorador

Abre la carpeta, haz clic derecho y selecciona Abrir en Terminal.

terminal
cd Desktop/mi-proyecto # Confirmar en qué carpeta estás: pwd
Resultado esperado
/Users/tunombre/Desktop/mi-proyecto
4

Inicializar el repositorio — git init

Este comando convierte tu carpeta normal en un repositorio de Git:

terminal
git init
Resultado esperado
Initialized empty Git repository in /Users/tunombre/Desktop/mi-proyecto/.git/
Tip: Esto crea una carpeta oculta llamada .git/ dentro de tu proyecto. Ahí Git guarda todo el historial. Nunca la borres.
5

Revisar el estado — git status

Antes de agregar archivos, siempre es bueno ver en qué estado está todo:

terminal
git status
Resultado esperado (primera vez)
On branch main

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)
        index.html
        style.css
        script.js

nothing added to commit but untracked files present
Tip: Los archivos en rojo son "untracked" — Git los ve pero aún no los rastrea. En el siguiente paso los agregaremos.
6

Agregar archivos al staging — git add

El staging area es una zona de preparación antes de guardar los cambios.

terminal
# Agregar todos los archivos de una vez: git add . # O agregar un archivo específico: git add index.html
git status después del add
Changes to be committed:
        new file:   index.html
        new file:   style.css
        new file:   script.js
Tip: Ahora los archivos aparecen en verde — están listos para el commit.
7

Primer commit — git commit

Un commit es una fotografía del estado de tu proyecto en este momento. Siempre lleva un mensaje descriptivo:

terminal
git commit -m "primer commit: estructura inicial del proyecto"
Resultado esperado
[main (root-commit) a3f2c1d] primer commit: estructura inicial del proyecto
 3 files changed, 45 insertions(+)
 create mode 100644 index.html
 create mode 100644 style.css
 create mode 100644 script.js
Buenos mensajes de commit: Deben describir QUÉ cambiaste.
✅ "agrega formulario de contacto"    ✅ "corrige bug en el menú móvil"
❌ "cambios"    ❌ "arreglé cosas"
8

Crear el repositorio en GitHub

Ahora necesitas un lugar en la nube donde vivir tu proyecto:

1
github.com → New repositoryInicia sesión y haz clic en "New repository".
2
Repository namePon el nombre de tu proyecto (sin espacios, usa guiones).
3
Public / PrivateElige la visibilidad según prefieras.
4
Create repositorySin marcar ninguna opción extra — el repo debe quedar completamente vacío.
Importante: NO marques "Add a README file" ni ninguna otra opción. El repo debe quedar completamente vacío.
9

Conectar tu proyecto con GitHub

Después de crear el repo vacío, GitHub te mostrará instrucciones. Usa la sección "…or push an existing repository from the command line" y ejecuta estos comandos:

terminal
# Agregar el origen remoto: git remote add origin https://github.com/tu-usuario/mi-proyecto.git # Renombrar la rama principal a main: git branch -M main
Tip: branch -M main asegura que tu rama se llame main, el estándar actual de GitHub.
10

Autenticarte con token de acceso

Nota: Salta al paso 11. Si durante el push te pide autenticación, regresa aquí.

GitHub ya no acepta contraseñas directas desde la terminal. Necesitas un Personal Access Token (PAT):

1
Foto de perfil → SettingsVe a tu configuración de cuenta en GitHub.
2
Developer settings → Personal access tokensEn el menú izquierdo, baja hasta Developer settings.
3
Generate new token (classic)Marca el scope ✅ repo — da acceso completo a repositorios.
4
Copiar el token inmediatamenteGitHub solo te lo muestra una vez. Guárdalo en un lugar seguro.
11

Hacer el push — git push

Sube tu proyecto a GitHub por primera vez:

terminal
git push -u origin main

Si te pide credenciales, ingresa tu usuario de GitHub y pega el token como contraseña:

Resultado esperado
Enumerating objects: 5, done.
Counting objects: 100% (5/5), done.
Writing objects: 100% (5/5), 1.23 KiB | 1.23 MiB/s, done.
To https://github.com/tu-usuario/mi-proyecto.git
 * [new branch]      main -> main
branch 'main' set up to track 'remote/origin/main'.
Tip: El flag -u guarda la conexión entre tu rama local y la remota. Los próximos pushes solo necesitan git push.
12

Verificar en GitHub

Ve a https://github.com/tu-usuario/mi-proyecto en tu navegador. Deberías ver todos tus archivos ahí.

🎉

¡Felicidades!

Tu proyecto ya vive en GitHub. Desde ahora tienes control de versiones, respaldo en la nube y puedes colaborar con otras personas.

Flujo continuo

Una vez que el repo está conectado, cada vez que hagas cambios el flujo es:

flujo de trabajo diario
# 1. Ver qué cambió git status # 2. Agregar los cambios git add . # 3. Guardar con descripción git commit -m "descripción de los cambios" # 4. Subir a GitHub git push

Comando Qué hace

git statusMuestra el estado actual de los archivos
